BookLa Ticketing Concert System

A simplified interface walkthrough simulation designed for a desktop GUI client environment.

Key Contributions & Individual Impact

While the baseline idea of building a ticketing solution was a collaborative team choice, I spearheaded the comprehensive technical execution, system architecture, and interface design of the final application. Inspired directly by professional modern ticketing ecosystems like the GoLive app, my focus was translating that smooth consumer mobile experience into a fully working VB.NET desktop framework.

  • UI/UX System Design: Architected the unified styling guidelines, layout structures, and data flows to mimic a modern digital ticketing agent.
  • Rigorous Testing & Optimization: Conducted extensive troubleshooting, debugging thousands of compilation errors to ensure absolute transaction stability and flawless multi-form data inheritance.

Main Form (Gateway Entry)

The system launcher landing screen. Designed to act as the consumer gateway, providing straightforward visual navigation buttons to explore available concert listings while keeping administrative access fields clean, secure, and restricted through a verified credential portal.

Admin Form (System Operations Panel)

The centralized control hub built strictly for event managers. Features modular structural blocks that separate concert event insertion (handling metadata updates such as date, location, and poster path browsing) from active transactional record management.

List of Concerts (Dynamic Event Feed)

A dynamically rendering layout structure presenting active show listings. Implemented to group rich imagery, distinct calendar times, and explicit location labels cleanly into scannable listing rows, improving immediate information tracking for users.

Book Ticket (Seating Tier Mapping)

The core seat allocation layout. Strongly inspired by clean stage maps used in GoLive, this system translates standard spatial layout rules into selectable tier zones (VIP, Premium, and Standard sections), passing context-relevant, real-time base rates instantly to local parameters.

Buy Ticket (Transactional Checkout Pipeline)

The system execution and mathematical processor. Tracks active item limits via operational condition limits, dynamically outputs subtotals combined with exact regional service taxes (6%), and sets up localized sequential seat indexes to handle asynchronous orders securely.

System Receipt (Final Order Generation)

The closing pipeline asset. Emits clean transactional receipts embedded with dynamically processed, random customer identifier numbers, formatted date strings, explicitly mapped line-by-line seating indexes, and formal execution parameters intended for local database commits.
